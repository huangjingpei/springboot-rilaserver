# 管理后台指南

本文档提供了有关如何设置和运行管理后台的说明。

## 设置

1.  **安装依赖项**

    在 `admin-ui` 目录下运行以下命令：

    ```bash
    npm install
    ```

2.  **配置环境变量**

    在 `admin-ui` 目录下创建一个 `.env.development` 文件，并添加以下内容：

    ```
    VITE_APP_BASE_API = '/api'
    ```

## 运行

在 `admin-ui` 目录下运行以下命令以启动开发服务器：

```bash
npm run dev
```

## 功能

### 1. 管理员登录

-   **路径**: `/login`
-   **描述**: 提供一个安全的登录页面，供管理员访问管理后台。

#### 默认登录凭据

**管理员账户:**
- **用户名**: `admin@example.com`
- **密码**: `password123`
- **权限**: 管理员权限，可访问所有功能

**测试用户账户:**
- **用户名**: `test@example.com`
- **密码**: `password123`
- **权限**: 普通用户权限

> **注意**: 
> - 首次部署后，请立即修改默认密码以确保系统安全
> - 生产环境中请删除或禁用测试账户
> - 密码要求至少6位字符，建议包含字母、数字和特殊字符

### 2. 仪表盘

-   **路径**: `/dashboard`
-   **描述**: 系统概览页面，显示关键统计信息和系统状态
-   **功能**:
    - 用户统计（总用户数、在线用户数）
    - 应用统计（总应用数、活跃应用数）
    - 流媒体统计（活跃流数量）
    - 系统状态监控（CPU、内存、磁盘使用率）
    - 最近活动记录

### 3. 用户管理

-   **路径**: `/users`
-   **描述**: 完整的用户管理功能，支持用户的增删改查操作
-   **功能**:
    - 用户列表展示和分页
    - 用户搜索和状态筛选
    - 添加新用户
    - 编辑用户信息
    - 删除用户
    - 用户状态管理（激活/禁用）

### 4. 应用管理

-   **路径**: `/apps`
-   **描述**: 应用程序管理界面，支持应用的生命周期管理
-   **功能**:
    - 应用列表展示
    - 应用搜索和筛选
    - 创建新应用
    - 编辑应用配置
    - 删除应用
    - 应用控制（启动/停止/重启）

### 5. 流媒体管理

-   **路径**: `/streams`
-   **描述**: 流媒体管理界面，管理所有直播流
-   **功能**:
    - 流媒体列表展示
    - 流状态监控
    - 流密钥生成和管理
    - 流URL复制
    - 流控制（启动/停止）
    - 流搜索和筛选

### 6. 网络测试

-   **路径**: `/network`
-   **描述**: 网络连接测试工具
-   **功能**:
    - 连接测试（指定地址和端口）
    - Ping测试
    - 测试结果时间线展示
    - 网络状态监控

### 7. 系统设置

-   **路径**: `/settings`
-   **描述**: 系统配置和维护功能
-   **功能**:
    - 基本设置（系统名称、描述、时区等）
    - 安全设置（密码策略、登录限制等）
    - 流媒体设置（端口配置、录制设置等）
    - 系统维护（数据库清理、优化、备份）
     - 系统信息查看

## 使用说明

### 首次登录

1. 启动后端服务器（Spring Boot应用）
2. 启动前端开发服务器：`npm run dev`
3. 访问 `http://localhost:5173/login`
4. 使用默认管理员账户登录：
   - 用户名：`admin@example.com`
   - 密码：`password123`

### 安全建议

1. **修改默认密码**: 首次登录后立即修改默认密码
2. **删除测试账户**: 生产环境中删除或禁用测试账户
3. **定期备份**: 使用系统设置中的备份功能定期备份数据
4. **监控日志**: 定期查看系统日志，及时发现异常

### 故障排除

#### 登录问题
- 确认后端服务器正在运行
- 检查数据库连接是否正常
- 验证用户名和密码是否正确

#### 功能异常
- 检查浏览器控制台是否有错误信息
- 确认API接口是否正常响应
- 查看后端日志文件

#### 性能问题
- 使用系统设置中的数据库优化功能
- 定期清理过期日志
- 监控系统资源使用情况
